# 第 2 章：从一个理想神经元走向有差异、有噪声的群体

对应实验：[lessons/02_neuron_group.py](../lessons/02_neuron_group.py)

输出图片：[outputs/02_neuron_group.png](../outputs/02_neuron_group.png)

## 这一章解决什么问题

上一章的神经元每次运行都产生完全相同的脉冲。但真实神经系统中：

- 不同神经元接收的平均驱动不同。
- 离子通道、突触释放和背景网络会引入波动。
- 即使方程形式相同，参数和初始状态也可能不同。

本章要回答：

1. 如何用一个 `NeuronGroup` 表示许多神经元。
2. 如何让同一群体中的神经元拥有不同参数。
3. `xi` 白噪声在方程中是什么意思。
4. 为什么随机微分方程使用 Euler 方法。
5. 怎样用栅格图读取群体脉冲活动。

## 1. 群体不等于复制 100 份 Python 对象

代码：

```python
N = 100
neurons = NeuronGroup(N, equations, ...)
```

Brian2 不要求你写：

```python
neuron_0 = ...
neuron_1 = ...
...
```

`NeuronGroup` 用统一的方程描述整个群体，同时让某些状态和参数按神经元索引存储。

可以把它理解为一张表：

| 神经元编号 i | v | mu |
|---:|---:|---:|
| 0 | 不同 | 0.9 |
| 1 | 不同 | 0.9 |
| ... | ... | ... |
| 50 | 不同 | 1.1 |
| ... | ... | ... |
| 99 | 不同 | 1.1 |

方程只有一份，但 v 和 mu 可以是长度为 100 的数组。

## 2. 本章方程在表达什么

```text
dv/dt = (mu - v)/tau + sigma*xi/sqrt(tau)
```

它由确定性部分和随机部分组成：

```text
确定性漂移：(mu - v)/tau
随机波动：sigma*xi/sqrt(tau)
```

### 确定性漂移

```text
(mu - v)/tau
```

与上一章相似，只是平衡点从固定的 1.1 变成了每个神经元自己的 `mu`。

- `mu > 1`：平均平衡点高于阈值，神经元倾向持续放电。
- `mu < 1`：没有噪声时，v 会停在阈值下方。

### 随机波动

```text
sigma*xi/sqrt(tau)
```

`xi` 是 Brian2 提供的高斯白噪声符号。

直觉上，白噪声表示：

- 每个很短时间段都有新的随机扰动。
- 不同时间段的扰动彼此独立。
- 扰动均值为 0。

它不是“每次实验随机选一个固定参数”，而是仿真过程中持续变化的随机驱动。

## 3. 为什么白噪声不能直接写成 `+ randn()`

连续时间白噪声不是普通无量纲随机数。`xi` 的单位是：

```text
1 / sqrt(second)
```

因为 `dv/dt` 的单位是 `1/second`，随机项也必须最终匹配变化率的单位。

本章：

```text
sigma * xi / sqrt(tau)
```

其中：

- sigma 无量纲。
- xi 是 `1/sqrt(second)`。
- `sqrt(tau)` 是 `sqrt(second)`。

两者相除得到：

```text
1 / second
```

与 `dv/dt` 一致。

这正体现 Brian2 单位系统的价值：它会阻止你把量纲不一致的随机项悄悄塞进方程。

## 4. `sigma` 控制的不是平均驱动

```python
sigma = 0.18
```

sigma 决定随机扰动强度。

- sigma 接近 0：轨迹接近确定性指数过程。
- sigma 增大：轨迹上下波动更明显。
- sigma 太大：两组 mu 的差别可能被噪声淹没。

mu 与 sigma 的角色不同：

- mu 改变长期趋势和平均平衡点。
- sigma 改变围绕趋势的随机散布。

建模时要区分“个体平均不同”和“同一个体随时间波动”。

## 5. 方程声明为什么多了一行

```python
equations = """
dv/dt = (mu - v)/tau + sigma*xi/sqrt(tau) : 1 (unless refractory)
mu : 1
"""
```

第一行定义动态变量 v。

第二行：

```text
mu : 1
```

只声明参数 mu，没有 `dmu/dt`，所以 Brian2 不会自动更新它。

创建群体后，我们再赋值：

```python
neurons[:50].mu = 0.9
neurons[50:].mu = 1.1
```

这比在方程外写两套神经元模型更清楚，因为两组只有一个参数不同。

## 6. `(unless refractory)` 到底冻结了什么

方程末尾：

```text
(unless refractory)
```

表示神经元处于不应期时，不更新这一行对应的变量 v。

因此放电后的 2 ms 内：

- 阈值不会再次触发。
- v 也保持 reset 后的值 0。

这与上一章不同。上一章没有该标记，v 在不应期内仍继续演化。

为什么有两种选择？

- 某些简化模型把不应期视为“膜电位钳在重置值”。
- 某些模型只禁止发放，但仍允许内部状态变化。

Brian2 不替你决定生物假设，而是让你明确写出选择。

## 7. 为什么随机方程使用 `method="euler"`

代码：

```python
method="euler"
```

含有 `xi` 的方程是随机微分方程。上一章使用的 `exact` 解析更新不能直接照搬。

Euler 方法的基本想法是：

```text
下一步状态
≈ 当前状态
  + 确定性变化率 * dt
  + 本时间步的随机增量
```

对于确定性方程，普通 Euler 近似为：

```text
v(t+dt) = v(t) + f(v,t)*dt
```

随机项的尺度与 `sqrt(dt)` 有关，Brian2 会根据 `xi` 的定义正确生成随机增量。你不应该手动再乘一次随意的 dt。

随机仿真对时间步尤其敏感。改变 dt 后，需要检查：

- 平均放电率是否稳定。
- 组间差异是否稳定。
- 轨迹分布是否明显变化。

## 8. 随机种子为什么重要

```python
seed(2)
```

随机模型每次运行本来可以产生不同结果。

固定种子后，随机数生成器会从同一个初始状态开始，因此在相同环境和配置下更容易复现实验。

固定种子不代表：

- 噪声消失了。
- 所有神经元获得相同随机数。
- 结果成为理论真值。

它只表示这一次伪随机序列可以重复。

科研中通常需要同时报告：

- 单个固定种子用于调试和图示。
- 多个种子或多次重复用于统计结论。

## 9. 初始状态为何也要随机

```python
neurons.v = "rand()"
```

字符串赋值不是把同一个随机数复制 100 次。Brian2 会按神经元索引计算表达式，每个神经元得到一个 0 到 1 之间的初始值。

如果所有神经元初始值相同、参数相同、噪声也不存在，它们可能完全同步。随机初始值可以打破这种人为同步。

两种写法含义不同：

```python
neurons.v = 0.5
```

所有神经元都设为 0.5。

```python
neurons.v = "rand()"
```

每个神经元独立求值。

## 10. 子组切片不是复制

```python
neurons[:50].mu = 0.9
neurons[50:].mu = 1.1
```

`neurons[:50]` 指向原群体中的 0 到 49 号神经元。

`neurons[50:]` 指向 50 到 99 号神经元。

它们是子组视图，不是重新创建的独立神经元。因此：

- SpikeMonitor 仍可一次记录整个群体。
- 神经元编号仍保持 0 到 99。
- 两组共享同一个时钟和方程。

## 11. 运行前先做定性预测

低驱动组：

```text
mu = 0.9 < threshold 1
```

没有噪声时不会放电。但噪声可能偶尔把 v 推过阈值。

高驱动组：

```text
mu = 1.1 > threshold 1
```

即使噪声均值为 0，它们总体上也更容易达到阈值。

因此预测：

- 两组都可能放电。
- 高驱动组总脉冲数更多。
- 每个神经元的具体脉冲时刻不规则。

实验输出示例：

```text
低驱动组脉冲数: 220
高驱动组脉冲数: 434
```

这支持定性预测，但单次运行的精确数字不是普遍常数。

## 12. 栅格图应该怎样读

绘图：

```python
plt.scatter(spikes.t / ms, spikes.i, s=4)
plt.axhline(50, color="tab:red", linestyle="--")
```

每个点代表一次脉冲：

- 横坐标：脉冲发生时间。
- 纵坐标：放电神经元编号。

红线以下是 0 到 49 号低驱动组，以上是 50 到 99 号高驱动组。

不要把点之间连成曲线。脉冲事件没有“从神经元 3 移动到神经元 4”的含义。

观察重点：

1. 上半区是否总体更密。
2. 是否存在所有神经元同时放电的垂直条纹。
3. 活动密度是否随时间发生系统变化。

本例没有神经元间连接，因此不应该把局部同步解释为网络传播。

## 13. 统计脉冲数时发生了什么

```python
sum(spikes.count[:50])
```

`spikes.count` 是长度为 100 的数组，每一项是对应神经元的脉冲数。

切片后求和得到整个子组总脉冲数。

如果要比较不同规模群体，更合理的量往往是平均放电率：

```text
平均放电率
= 总脉冲数 / 神经元数量 / 仿真时长
```

因为总脉冲数会随群体大小和实验时长自然增大。

## 14. 本章没有模拟什么

本章中的神经元彼此独立：

- 没有突触。
- 没有共同输入。
- 没有网络反馈。

它展示的是“参数和噪声怎样影响群体统计”，不是网络动力学。

这一区分非常重要。看到 100 个神经元同时出现在一张图上，不代表它们已经构成相互作用的网络。

## 15. 常见误区

### 误区一：mu=0.9 的神经元绝对不会放电

确定性平衡点低于阈值，但噪声可以造成阈值穿越。

### 误区二：固定 seed 后模型就没有随机性

随机轨迹仍然存在，只是可以重复。

### 误区三：一个种子的脉冲数就是结论

随机模型应通过多个种子、均值和置信区间评估稳定性。

### 误区四：`rand()` 与 `xi` 是同一种随机性

- `rand()` 在赋值时生成随机初始状态。
- `xi` 在整个仿真中持续提供白噪声。

### 误区五：高驱动组每个神经元都一定比低驱动组放得多

群体平均趋势不保证每个个体的排序。噪声和初始状态可能造成重叠。

## 16. 动手实验

### 实验 A：关闭噪声

设置：

```python
sigma = 0
```

预测：

- 低驱动组不再放电。
- 高驱动组趋向更规则的周期放电。
- 随机初始状态只影响早期相位。

### 实验 B：逐渐增大噪声

测试：

```text
sigma = 0.05
sigma = 0.18
sigma = 0.5
```

记录两组平均放电率，并观察组间差异是否被噪声缩小。

### 实验 C：交换两组 mu

把前 50 个设为 1.1，后 50 个设为 0.9。

预测：栅格图密集区域应从红线上方移动到下方。

这能验证图上的纵轴确实对应神经元编号，而不是某种放电强度。

### 实验 D：多种子重复

循环使用 10 个种子，每个种子记录两组平均放电率。计算：

- 均值。
- 标准差。
- 高驱动组放电率高于低驱动组的次数。

这个实验开始把“看一张图”升级为统计判断。

### 实验 E：比较冻结与不冻结不应期

删除 `(unless refractory)`，比较同一组参数下的放电率。

先解释你预测差异来自哪里，再运行。

## 本章小结

本章把模型复杂性拆成了三类：

```text
共同结构：所有神经元使用同一条微分方程
个体差异：每个神经元可以拥有不同 mu 和初始 v
时间波动：xi 在仿真过程中持续引入随机扰动
```

你还应该记住：

- 群体统计与单个神经元行为不是同一层次。
- 随机模型的单次运行不是最终结论。
- 栅格图记录的是离散脉冲，不是连续膜电位。

下一章会第一次让神经元之间发生作用：一个源神经元的脉冲经过突触权重和延迟，改变另一个神经元的状态。
