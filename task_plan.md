## 目标

创建一套中文、可运行、循序渐进的 Brian2 教程项目。学习者完成后，应能独立建立脉冲神经网络模型、记录和分析结果、组织复杂仿真，并理解 Brian2 的性能优化与代码生成能力。

## 范围

- 安装、单位系统与最小仿真
- `NeuronGroup` 方程、阈值、重置、不应期和随机性
- `Synapses` 连接、延迟、事件、突触可塑性
- `SpikeGeneratorGroup`、`PoissonGroup`、`TimedArray` 等输入
- `StateMonitor`、`SpikeMonitor`、`PopulationRateMonitor`
- `Network`、`store/restore`、调度与自定义操作
- 多区室模型与 `SpatialNeuron`
- 方程、函数、命名空间、子表达式和事件驱动变量
- NumPy/Cython/C++ standalone 代码生成与性能
- 调试、可复现性、常见错误和实验组织
- 一个综合项目

## 决策

- 文件夹名使用中文 `Brian2完整教程`，示例文件名使用 ASCII，便于命令行操作。
- 教程以独立 `.py` 文件为主，减少 Notebook 环境依赖。
- 对“完整功能”采用功能地图加代表性实例，而不是机械罗列全部 API。
- 当前 Python 3.11 锁定 Brian2 2.9.0；文档以稳定版 2.10.1 的功能分类作为全景参考。

## 错误记录

| 错误 | 尝试 | 解决 |
|---|---|---|
| 首次依赖安装命令超过 120 秒 | 1 | 检查后台进程并等待完成；Brian2 2.9.0 安装成功 |
| `pip show matplotlib` 在 GBK 控制台输出许可证时触发 UnicodeEncodeError | 1 | 仅为展示输出编码问题；后续命令启用 `PYTHONUTF8=1` |
| Brian2 2.9.0 与 pip 选取的 NumPy 2.4.6 不兼容，导入时报 `ndarray.ptp` 缺失 | 1 | 将 NumPy 锁定为 1.26.4 |
| NumPy 后端运行 `SpatialNeuron` 时缺少 SciPy | 1 | 添加 SciPy 1.14.1 依赖 |
| 综合平衡网络外部输入过弱，运行成功但没有脉冲 | 1 | 提高泊松输入频率与单次权重后重新验证 |
| 递归统计文件时扫描 `.venv` 超过 10 秒 | 1 | 该统计不影响教程，不再扫描虚拟环境 |
| Windows 下 `py_compile` 未展开 `lessons\*.py` 通配符 | 1 | 改为由 PowerShell 枚举文件后传入 |
